# GitHub Global - 保姆级本地运行指南

本文档手把手带你从零开始，在本地运行 GitHub Global 项目。按照步骤来，不会跳过任何细节。

---

## 前置条件

请确保你的电脑上已经安装了以下软件：

| 软件 | 最低版本 | 检查命令 | 安装地址 |
|---|---|---|---|
| Node.js | 20.0.0 | `node -v` | [nodejs.org](https://nodejs.org/en) |
| npm | 10.0.0 | `npm -v` | 随 Node.js 一起安装 |
| MySQL | 8.0 | `mysql --version` | [dev.mysql.com](https://dev.mysql.com/downloads/) |
| Git | 任意版本 | `git --version` | [git-scm.com](https://git-scm.com/) |

> **Windows 用户注意**：推荐使用 **Git Bash** 来运行以下所有命令，避免命令兼容性问题。

---

## 第一步：克隆项目

```bash
git clone https://github.com/liyupi/github-global.git
cd github-global
```

---

## 第二步：安装依赖

```bash
npm install
```

安装完成后，目录下会出现 `node_modules` 文件夹。如果安装失败，可以尝试：

```bash
# 清理缓存后重试
rm -rf node_modules package-lock.json
npm install
```

---

## 第三步：创建 MySQL 数据库

### 3.1 启动 MySQL 服务

- **macOS**（Homebrew 安装）：`brew services start mysql`
- **Windows**：在「服务」中启动 MySQL，或使用 MySQL 安装器自带的启动方式
- **Linux**：`sudo systemctl start mysql`

### 3.2 创建数据库

用任意 MySQL 客户端（命令行 / Navicat / IDEA 数据库插件等）连接本地 MySQL，执行：

```sql
CREATE DATABASE github_global CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> 如果你修改了 MySQL 的用户名或密码，记住它们，后续配置环境变量时需要用到。

---

## 第四步：创建 GitHub App

这是本项目的**核心前置配置**，不创建 GitHub App 就无法登录和使用。

### 4.1 进入创建页面

登录 GitHub，访问：[https://github.com/settings/apps/new](https://github.com/settings/apps/new)

### 4.2 填写基本信息

| 字段 | 填写内容 |
|---|---|
| **GitHub App name** | 随便取一个名字（如 `my-github-global`），全局唯一 |
| **Homepage URL** | `http://localhost:3123` |
| **Callback URL** | `http://localhost:3123/api/auth/callback/github` |
| **Setup URL**（可选） | 留空即可 |
| **Webhook** | **取消勾选** Active（本地开发暂不需要） |

> 注意端口是 **3123**，不是默认的 3000。

### 4.3 配置权限

展开 **Permissions** 部分，设置以下权限：

**Repository permissions（仓库权限）：**

| 权限 | 级别 | 用途 |
|---|---|---|
| **Contents** | Read & Write | 读取仓库文件、写入翻译文件 |
| **Metadata** | Read-only | 读取仓库基本信息 |
| **Pull requests** | Read & Write | 创建翻译完成的 PR |
| **Webhooks** | Read & Write | 自动翻译功能需要（可暂时不设） |

**Account permissions（账户权限）：**

| 权限 | 级别 | 用途 |
|---|---|---|
| **Email addresses** | Read-only | 获取用户邮箱 |

### 4.4 设置安装范围

在页面底部 **Where can this GitHub App be installed?** 选择：

- **Any account** — 所有用户都能安装

### 4.5 创建并获取密钥

点击 **Create GitHub App** 按钮创建成功后：

1. **记录 App ID**：页面最上方的 `App ID`
2. **记录 Client ID**：页面中的 `Client ID`（格式如 `Iv23li...`）
3. **生成 Client Secret**：点击 `Generate a new client secret`，**立即复制保存**（只显示一次）
4. **生成 Private Key**：点击 `Generate a private key`，浏览器会下载一个 `.pem` 文件
5. **记录 App Slug**：就是你的 GitHub App 名称的小写连字符版本（如 `my-github-global`），可以在 App 页面的 URL 中看到：`https://github.com/settings/apps/my-github-global`

### 4.6 放置私钥文件

将下载的 `.pem` 私钥文件重命名为 `private-key.pem`，放到项目根目录下：

```
github-global/
├── private-key.pem    ← 放在这里
├── package.json
├── ...
```

> 这个文件已经在 `.gitignore` 中，不会被提交到 Git。

## 第五步：获取 OpenRouter API Key

项目使用 [OpenRouter](https://openrouter.ai/) 统一调用各种 AI 大模型进行翻译。

1. 访问 [https://openrouter.ai/](https://openrouter.ai/)，注册并登录
2. 进入 [API Keys 页面](https://openrouter.ai/keys)，点击 **Create Key**
3. 复制生成的 API Key（格式如 `sk-or-v1-xxxx...`）

> 新注册用户通常有免费额度，足够本地测试使用。

## 第六步：配置环境变量

### 6.1 创建配置文件

项目同时需要 `.env` 和 `.env.local` 两个环境变量文件：

```bash
cp .env.example .env
cp .env.example .env.local
```

### 6.2 填写配置

用编辑器打开 `.env` 和 `.env.local`（两个文件内容保持一致），按下面的说明填写：

```bash
# ===== 数据库 =====
# 格式：mysql://用户名:密码@地址:端口/数据库名
# 如果你的 MySQL root 密码是 123456，就改成：
DATABASE_URL=mysql://root:123456@localhost:3306/github_global

# ===== NextAuth 认证 =====
# 本地开发地址（注意端口是 3123）
NEXTAUTH_URL=http://localhost:3123
# 随便填一个 32 位以上的字符串，用于加密 Session
AUTH_SECRET=this-is-a-random-secret-string-at-least-32-chars

# ===== GitHub App 配置 =====
# 第四步中获取的信息，逐一填入
GITHUB_APP_ID=你的AppID
GITHUB_APP_CLIENT_ID=你的ClientID
GITHUB_APP_CLIENT_SECRET=你的ClientSecret
GITHUB_APP_PRIVATE_KEY_PATH=./private-key.pem
GITHUB_APP_SLUG=你的app名称slug

# ===== Webhook（本地开发可不填） =====
GITHUB_WEBHOOK_SECRET=

# ===== OpenRouter =====
PLATFORM_OPENROUTER_API_KEY=sk-or-v1-你的key
```

### 6.3 两个文件的关系

| 文件 | 读取方 | 说明 |
|---|---|---|
| `.env` | Prisma / 后端代码 | 数据库连接、GitHub App 配置 |
| `.env.local` | Next.js / NextAuth | 认证相关配置（NextAuth 只读此文件） |

> 为了避免配置遗漏，**建议两个文件内容保持完全一致**。

## 第七步：初始化数据库

运行以下两条命令，Prisma 会根据 `schema.prisma` 自动在 MySQL 中创建所有表：

```bash
# 生成 Prisma 客户端代码
npx prisma generate

# 将表结构推送到数据库
npx prisma db push
```

成功后会看到类似输出：

```
🚀  Your database is now in sync with your Prisma schema.
```

> 如果报连接错误，请检查 MySQL 是否已启动、DATABASE_URL 中的用户名密码是否正确。

可以用以下命令打开数据库可视化管理界面，验证表是否创建成功：

```bash
npx prisma studio
```

```bash
npm run dev
```

看到以下输出，说明启动成功：

```
▲ Next.js 15.x.x
- Local:        http://localhost:3123
✓ Ready in xxxms
```

打开浏览器，访问 **http://localhost:3123** ，你应该能看到项目主页。

## 第九步：安装 GitHub App 并登录

### 9.1 点击登录

在页面右上角点击 **GitHub 登录** 按钮，会跳转到 GitHub 授权页面。

### 9.2 授权登录

在 GitHub 页面上确认授权，授权成功后会跳转回项目。

### 9.3 安装 GitHub App

登录成功后，页面会引导你安装 GitHub App（弹出 GitHub 的安装页面）。选择你要授权访问的仓库：

- **All repositories**：授权所有仓库
- **Only select repositories**：只授权指定仓库

安装完成后，就可以正常使用所有功能了。

## 第十步：验证核心功能

按以下流程验证项目是否正常运行：

1. **导入仓库**：在仪表盘页面输入一个 GitHub 仓库地址（建议先用一个文件较少的测试仓库），点击导入
2. **配置翻译**：进入仓库详情，选择目标语言（如英语 `en`），勾选要翻译的文件
3. **执行翻译**：点击「开始翻译」，观察任务进度
4. **查看结果**：翻译完成后，到 GitHub 仓库中查看是否生成了 PR

## Docker 方式运行（可选）

如果你更习惯用 Docker，项目也支持容器化部署：

```bash
# 1. 确保 private-key.pem 已放到项目根目录
# 2. 确保 .env 文件已配置好
# 3. 启动服务（包含 MySQL + 应用）
docker-compose up -d

# 4. 初始化数据库
docker-compose exec app npx prisma db push

# 5. 查看日志
docker-compose logs -f app

# 6. 停止服务
docker-compose down
```

## 常见问题排查

### Q1：npm install 失败

```bash
# 清理缓存重试
rm -rf node_modules package-lock.json
npm install

# 如果网络问题，可以使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### Q2：数据库连接失败

依次检查：
1. MySQL 服务是否已启动
2. `.env` 文件中的 `DATABASE_URL` 用户名和密码是否正确
3. 数据库 `github_global` 是否已创建
4. 防火墙是否拦截了 3306 端口

### Q3：GitHub 登录后报错 500

- 检查 `.env.local` 文件是否存在并且配置正确（NextAuth 只读 `.env.local`）
- 确认 `GITHUB_APP_CLIENT_ID` 和 `GITHUB_APP_CLIENT_SECRET` 填写正确
- 确认 GitHub App 的 Callback URL 设置为 `http://localhost:3123/api/auth/callback/github`

### Q4：Private Key 读取失败

- 确认 `private-key.pem` 文件在项目根目录下
- 确认文件内容完整（以 `-----BEGIN RSA PRIVATE KEY-----` 开头、`-----END RSA PRIVATE KEY-----` 结尾）
- 确认 `.env` 中 `GITHUB_APP_PRIVATE_KEY_PATH=./private-key.pem`

### Q5：翻译任务失败

- 查看终端控制台的错误日志，确认具体错误原因
- **401 Unauthorized**：OpenRouter API Key 无效或余额不足
- **403 Forbidden**：所选的 AI 模型在当前地区不可用，换一个模型试试
- **422 sha wasn't supplied**：目标文件已存在但代码未获取 SHA，通常重新执行翻译即可

### Q6：页面样式异常 / 白屏

```bash
# 清理 Next.js 缓存后重启
rm -rf .next
npm run dev
```

### Q7：端口 3123 被占用

```bash
# macOS / Linux 查看占用进程
lsof -i :3123

# Windows 查看占用进程
netstat -ano | findstr :3123

# 终止进程后重新启动
```

### Q8：登录后无法操作仓库（权限不足）

说明 GitHub App 还未安装到你的账户：
1. 进入 GitHub **Settings → Applications → GitHub Apps**
2. 找到你创建的 App，点击 **Configure**
3. 选择要授权的仓库，保存

如果仍然有问题，可以先取消安装（Uninstall），再重新安装。

## 进阶：开启自动增量翻译

自动增量翻译需要 GitHub 能够访问你的本地服务，因此需要**内网穿透**。

### 1. 安装 Ngrok

到 [ngrok.com](https://ngrok.com/) 下载安装，并绑定你的 API 密钥：

```bash
ngrok config add-authtoken 你的AuthToken
```

### 2. 启动内网穿透

```bash
ngrok http 3123
```

会得到一个公网地址，类似 `https://xxxx-xxx.ngrok-free.app`。

### 3. 更新配置

将 `.env` 和 `.env.local` 中的 `NEXTAUTH_URL` 改为公网地址：

```bash
NEXTAUTH_URL=https://xxxx-xxx.ngrok-free.app
```

同时到 GitHub App 设置页面，更新：
- **Callback URL** → `https://xxxx-xxx.ngrok-free.app/api/auth/callback/github`
- 勾选 **Webhook Active**，填写 **Webhook URL** → `https://xxxx-xxx.ngrok-free.app/api/webhooks/github`
- 填写 **Webhook Secret**（与 `.env` 中的 `GITHUB_WEBHOOK_SECRET` 保持一致）

### 4. 配置 Webhook 权限

在 GitHub App 设置页面：
- **Repository permissions → Webhooks**：设为 `Read & Write`
- **Subscribe to events**：勾选 `Push`

保存后重新安装 GitHub App，在项目的翻译配置页面开启「自动翻译」开关即可。
