# Github Translator - 快速启动说明

本文档说明如何快速启动 Github Translator 项目。

---

## 前置要求

在开始之前，请确保已完成：

1. ✅ [人工配置文档](./人工配置文档.md) 中的所有配置步骤
2. ✅ 安装 Node.js 20+ 和 npm 10+
3. ✅ 安装 MySQL 8.0+
4. ✅ 配置好 `.env` 文件

---

## 方式一: 本地开发启动

### 1. 安装依赖

```bash
npm install
```

### 2. 生成 Prisma Client

```bash
npm run db:generate
```

### 3. 初始化数据库

#### 方式 A: 使用 Prisma Migrate（推荐）

```bash
npm run db:migrate
```

这会创建数据库迁移文件并应用到数据库。

#### 方式 B: 直接推送 Schema

```bash
npm run db:push
```

这会直接将 Prisma Schema 同步到数据库（适合开发环境）。

### 4. 启动开发服务器

```bash
npm run dev
```

### 5. 访问应用

打开浏览器访问: http://localhost:3000

---

## 方式二: Docker 部署

### 1. 配置环境变量

确保 `.env` 文件已正确配置。

### 2. 启动 Docker Compose

```bash
docker-compose up -d
```

这会启动两个容器：
- `app`: Next.js 应用
- `db`: MySQL 数据库

### 3. 运行数据库迁移

```bash
docker-compose exec app npx prisma migrate deploy
```

### 4. 查看日志

```bash
docker-compose logs -f app
```

### 5. 访问应用

打开浏览器访问: http://localhost:3000

### 6. 停止服务

```bash
docker-compose down
```

### 7. 清理数据（可选）

```bash
docker-compose down -v  # 删除数据卷
```

## 使用流程

### 1. 登录系统

1. 访问 http://localhost:3000
2. 点击"使用 GitHub 登录"
3. 授权 GitHub App

### 2. 导入仓库

使用 API 导入仓库：

```bash
curl -X POST http://localhost:3000/api/repos \
  -H "Content-Type: application/json" \
  -d '{
    "repoUrl": "https://github.com/username/repo-name"
  }'
```

### 3. 配置翻译

更新仓库配置：

```bash
curl -X PUT http://localhost:3000/api/repos/{repo_id}/config \
  -H "Content-Type: application/json" \
  -d '{
    "baseLanguage": "zh-CN",
    "targetLanguages": ["en", "ja", "ko"]
  }'
```

### 4. 创建翻译任务

```bash
curl -X POST http://localhost:3000/api/translations \
  -H "Content-Type: application/json" \
  -d '{
    "repositoryId": "{repo_id}",
    "targetLanguages": ["en", "ja"],
    "type": "FULL"
  }'
```

### 5. 查看翻译进度

使用 SSE 实时获取进度：

```javascript
const eventSource = new EventSource('/api/translations/{task_id}/progress');

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('进度:', data.progress);
};
```

### 6. 查看 Pull Request

翻译完成后，会自动创建 Pull Request，可以在 GitHub 仓库中查看和合并。

## 故障排查

### 问题 1: 端口被占用

**错误**: `Error: listen EADDRINUSE: address already in use :::3000`

**解决方案**:

```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# macOS/Linux
lsof -ti:3000 | xargs kill -9
```

### 问题 2: 数据库连接失败

**错误**: `Can't reach database server at localhost:3306`

**解决方案**:

1. 检查 MySQL 是否启动
2. 检查 `.env` 中的 `DATABASE_URL` 配置
3. 测试数据库连接：

```bash
mysql -h localhost -u github_translator -p
```

### 问题 3: Prisma Client 未生成

**错误**: `Cannot find module '@prisma/client'`

**解决方案**:

```bash
npm run db:generate
```

### 问题 4: GitHub App 认证失败

**错误**: `Error: Invalid GitHub App credentials`

**解决方案**:

1. 检查 `.env` 中的 GitHub App 配置
2. 确保私钥格式正确
3. 检查 App ID 和 Client ID 是否匹配

### 问题 5: OpenRouter API 调用失败

**错误**: `401 Unauthorized` 或 `429 Rate Limit`

**解决方案**:

1. 检查 API Key 是否正确
2. 检查 OpenRouter 账户余额
3. 检查是否超过 API 限流

## 性能优化建议

### 1. 数据库优化

- 定期清理旧的翻译任务记录
- 为常用查询添加索引
- 使用连接池

### 2. 翻译队列优化

- 调整并发数（默认 3）
- 使用 Redis 替代本地队列（生产环境）

### 3. API 缓存

- 缓存语言列表和模型列表
- 缓存仓库文件树（短时间）

## 生产环境部署建议

### 1. 环境变量

- 使用生产环境的域名更新 `NEXTAUTH_URL`
- 使用强密码和密钥
- 不要在代码中硬编码敏感信息

### 2. 数据库

- 使用云数据库服务（如 AWS RDS、阿里云 RDS）
- 配置数据库备份
- 启用 SSL 连接

### 3. 反向代理

使用 Nginx 或 Caddy 作为反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4. HTTPS

使用 Let's Encrypt 配置 SSL 证书：

```bash
certbot --nginx -d your-domain.com
```

### 5. 监控和日志

- 配置日志收集（如 Winston、Pino）
- 配置错误监控（如 Sentry）
- 配置性能监控（如 New Relic）
